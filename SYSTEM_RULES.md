# System Global Rules
이 규칙은 **LLM 기반 금융 데이터 리포팅 파이프라인** 아키텍처의 핵심을 이루며, 모든 개발과 코드 작성 과정에서 **최우선**으로 지켜야 합니다.

- **Rule 1 (안전성)**: LLM(OpenAI 등)은 절대로 이 시스템 내에서 독립적인 직접 투자 판단(매수/매도 결정)을 내리지 않습니다. LLM은 오직 두 가지 역할만 수행합니다: (1) 뉴스 텍스트를 분석하여 `{"sentiment_score": float}` 형태(-1.0 ~ 1.0)의 JSON 데이터를 반환하는 **감성 분석 계층**, (2) 백테스트 수치를 사람이 읽을 수 있는 Markdown으로 변환하는 **리포트 자동화 계층**.

- **Rule 2 (모듈화)**: 모든 코드는 전역 변수(global) 의존성을 완벽히 배제하며, 순수한 객체지향 프로그래밍(OOP) 패러다임의 클래스(Class) 기반으로 작성되어야 합니다. 의존성은 언제든 주입(Injection) 및 교체 가능하도록 느슨하게 설계합니다.

- **Rule 3 (예외 처리)**: API나 외부 라이브러리(yfinance, OpenAI, 외부 뉴스 스크래퍼 등)의 모든 외부 연결 및 데이터 파싱 시에는 타임아웃(Timeout)과 광범위한 `try-except` 예외 처리를 무조건 구현해야 합니다. 어플리케이션 크래시가 발생하는 것을 용납하지 않습니다.

- **Rule 4 (문서화 강제)**: 새로운 로직이 추가되거나 아키텍처 모듈이 갱신될 때마다, `docs/INTERVIEW_PREP.md` 및 `docs/portfolio_positioning.md`에 해당 기술의 선택 배경, 엔지니어링 Trade-off 관점에서의 득실, 그리고 핵심 작동 원리를 반드시 문서화합니다.

- **Rule 5 (포지셔닝 유지)**: 이 프로젝트는 투자 자문 또는 자동매매 성과 검증 시스템이 아닙니다. README, 코드 docstring, 텔레그램 메시지, 대시보드 UI 문구에서 "자동매매 봇", "투자 추천", "수익 극대화", "실전 매매 엔진" 등의 표현을 사용하지 않습니다. KIS API 주문 실행은 항상 optional/paper execution 영역으로 설명합니다.
