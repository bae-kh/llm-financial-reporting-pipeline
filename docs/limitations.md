# Limitations & Disclaimer
### LLM 기반 금융 데이터 리포팅 파이프라인 — 한계 및 면책 조항

> 이 문서는 이 프로젝트를 사용하거나 평가하는 모든 분이 반드시 읽어야 합니다.

---

## ⚠️ 핵심 면책 조항

**이 프로젝트는 투자 자문, 자동매매 성과 검증, 또는 수익 보장 서비스가 아닙니다.**  
이 시스템의 모든 출력물(리포트, 지표, 대시보드, 감성 점수)은 어떠한 투자 판단의 근거로도 사용할 수 없습니다.  
이 프로젝트는 **데이터 파이프라인과 LLM 리포트 자동화 workflow를 보여주기 위한 리포팅 자동화 프로토타입**입니다.

---

## 1. 투자 조언 아님

- 이 시스템은 투자 자문업에 해당하지 않으며, 어떠한 종목의 매수·매도를 추천하지 않습니다.
- LLM이 생성한 감성 점수와 리포트는 텍스트 분석 결과물이며 투자 판단 근거가 아닙니다.
- 이 시스템을 참고하여 발생한 투자 손실에 대해 이 프로젝트는 어떠한 책임도 지지 않습니다.

## 2. 백테스트는 과거 시뮬레이션

- 모든 백테스트 결과(CAGR, MDD, Sharpe, Alpha 등)는 **과거 데이터 기반 시뮬레이션**입니다.
- 과거 성과는 미래 수익의 지표가 아닙니다.
- 실제 시장에서는 슬리피지, 유동성 제약, 급격한 가격 변동, 거래 수수료 등 추가 요인이 존재합니다.
- 백테스트에서의 Look-ahead 편향 방어(`df.iloc[:i+1]`)와 거래비용(0.1%) 반영은 현실성을 높이기 위한 조치이나, 실제 거래 결과를 보장하지 않습니다.

## 3. LLM 감성 분석의 한계

- LLM 출력은 **비결정적(non-deterministic)**이며, 동일한 뉴스 텍스트에 대해 다른 감성 점수를 반환할 수 있습니다.
- 이를 방어하기 위해 3중 방어막(JSON 모드 강제, try-except fallback, Pydantic 검증)을 구현했으나 완전한 일관성을 보장하지 않습니다.
- LLM Hallucination: LLM이 사실과 다른 내용을 리포트에 포함할 수 있으며, 이를 완전히 제거할 수 없습니다.

## 4. 뉴스 데이터 편향

- Google News RSS는 특정 언론사에 편향될 수 있으며, 수집된 헤드라인이 시장 전체 심리를 대표하지 않습니다.
- 헤드라인만으로 감성을 분석하므로 기사 본문, 맥락, 시장 반응이 반영되지 않습니다.
- 뉴스 커버리지가 낮은 날(주말, 휴일)에는 중립값(0.0)이 사용됩니다.

## 5. KIS API 주문 기능

- KIS API를 통한 주문 기능은 **Optional / Paper Execution 실험 기능**입니다.
- 이 포트폴리오는 KIS API 주문 안정성이나 실전 거래 성과를 검증하는 프로젝트가 아닙니다.
- `KIS_ENVIRONMENT`는 기본값이 `"virtual"` (모의투자)입니다. 실전 전환은 명시적 설정 변경이 필요합니다.
- 실거래 손실에 대해 이 프로젝트는 어떠한 책임도 지지 않습니다.

## 6. 단일 종목 한계

- 현재 TSLA(Tesla) 단일 종목만 지원합니다.
- 단일 종목 집중으로 인한 종목 특수 리스크(Idiosyncratic Risk)가 존재합니다.
- 다중 종목 분산 포트폴리오 효과가 없습니다.

## 7. 기술적 한계

- 파라미터(익절/손절 기준, RSI 임계값, 감성 임계값)에 결과가 민감하게 반응할 수 있습니다.
- 몬테카를로 시뮬레이션의 감성 점수는 실제 뉴스 감성과 다를 수 있습니다.
- `requirements.txt` 버전 미고정으로 인해 라이브러리 업데이트 시 동작이 달라질 수 있습니다.

## 이 프로젝트의 목적

이 프로젝트는 다음 역량을 보여주기 위한 **리포팅 자동화 프로토타입**입니다:

- Python 데이터 파이프라인 설계
- LLM 기반 비정형 텍스트 분석 및 output 검증
- 백테스트와 리스크 지표 계산
- LLM을 활용한 Markdown 리포트 자동 생성
- EC2/Linux 배치 실행 및 Streamlit 대시보드

**투자 목적으로 사용하지 마십시오.**
